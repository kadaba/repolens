# app-classifier v0.5.0 — Design

**Status:** Approved (2026-05-22)
**Author:** ckadaba
**Scope:** Multi-provider LLM layer + `classify_smart()` rule-first/LLM-escalated classification + `map_code()` non-LLM impact analysis + comprehensive docs refresh

---

## Goals

1. Let users hook app-classifier into any LLM provider (OpenRouter, OpenAI, Anthropic, Ollama, or any OpenAI-compatible local server) with a tiny adapter layer.
2. Add `classify_smart()` — runs the deterministic classifier first; escalates to the agentic loop **only** when confidence is low.
3. Add `map_code()` — deterministic structural code mapping (file dependency graph for 6 languages + Python function-call graph) for impact analysis, with no LLM dependency.
4. Refresh README + docs so a new user can install, configure, and run end-to-end in under five minutes.

## Non-goals

- Java/JS/Go function-call graphs (deferred to v0.6.0)
- Replacing `classify_agentic()` — it stays as the lowest-level agentic API
- Vendoring official SDK clients — providers use stdlib `urllib`, with SDK-backed adapters available as optional extras
- Changing the existing `classify()` / `classify_agentic()` API surface

## Constraints

- **Zero runtime deps preserved.** All five LLM adapters MUST use stdlib `urllib.request` for HTTP. Official SDK adapters (if added) go into `pip install app-classifier[openai]`-style extras only.
- **Backwards compatibility.** All existing tests pass unchanged. The BYO-callable `llm_provider=fn` contract used by `classify_agentic()` continues to work.
- **No surprises in the default path.** `classify()` and `classify_smart()` (when no provider is configured) MUST never make network calls.

## Components

### 1. LLM provider layer (`app_classifier.llm`)

**Protocol (`provider.py`):**

```python
from typing import Protocol, runtime_checkable

@runtime_checkable
class LLMProviderProtocol(Protocol):
    name: str  # "openai", "anthropic", "openrouter", "ollama", "openai_compat"
    async def complete(
        self,
        prompt: str,
        *,
        max_tokens: int = 400,
        temperature: float = 0.0,
    ) -> str | None: ...
```

**Naming — collision avoidance:** `classifier.py:32` already exports `LLMProvider = Callable[..., Awaitable[Optional[str]]]` and it's imported in `tests/test_classifier.py`. The new Protocol is named `LLMProviderProtocol` to preserve the existing public symbol. The Callable alias `LLMProvider` is kept exported unchanged.

**Adapter ↔ Callable compatibility (load-bearing):** Every adapter class implements `async def complete(self, prompt, *, max_tokens=400, temperature=0.0) -> str | None`, **and** aliases `__call__ = complete` at class scope. Concrete shape:

```python
class OpenAIProvider:
    name = "openai"
    def __init__(self, api_key, model="gpt-4o-mini", base_url=None): ...
    async def complete(self, prompt, *, max_tokens=400, temperature=0.0):
        ...
    __call__ = complete  # satisfies the existing Callable contract in classify_agentic
```

This means `classify_agentic(llm_provider=OpenAIProvider(...))` calls `provider(prompt=..., max_tokens=..., temperature=...)` and gets routed to `complete()`. Both contracts are satisfied by a single implementation.

`complete()` returns `None` on transport failure / provider refusal. Callers (including `classify_agentic` at `agent.py:407` and `:535`) already handle `None` by stopping the loop cleanly.

**Adapters (`adapters/*.py`):**

| Adapter | Constructor | Endpoint shape |
|---|---|---|
| `OpenAIProvider` | `(api_key, model="gpt-4o-mini", base_url=None)` | OpenAI `/v1/chat/completions` |
| `AnthropicProvider` | `(api_key, model="claude-haiku-4-5-20251001")` | Anthropic `/v1/messages` |
| `OpenRouterProvider` | `(api_key, model="anthropic/claude-haiku-4-5-20251001")` | OpenAI-shape via `openrouter.ai/api/v1` + `HTTP-Referer` + `X-Title` headers |
| `OllamaProvider` | `(host="http://localhost:11434", model="llama3.2")` | Ollama `/api/generate` (non-streaming mode) |
| `OpenAICompatProvider` | `(base_url, api_key=None, model=...)` | OpenAI-shape against arbitrary base_url. Covers LM Studio, vLLM, llama.cpp server, Groq, Together, Fireworks, DeepSeek, Mistral, anything OpenAI-compatible. |

**HTTP layer (`_http.py`):** Single stdlib `urllib.request` wrapper. Handles JSON encode, timeout (30s default), retries (1 retry on transient 5xx). Sync function `post_json(url, body, headers) -> dict | None`. Async wrapper is `await asyncio.to_thread(post_json, ...)`.

**Config loader (`config.py`):**

Precedence (highest → lowest):
1. Explicit `load_provider(name="...")` argument
2. `APP_CLASSIFIER_LLM_PROVIDER` env var
3. `~/.app-classifier/providers.json` (or `$APP_CLASSIFIER_CONFIG` override)
4. Best-effort autodetect: if `ANTHROPIC_API_KEY` present → anthropic; elif `OPENAI_API_KEY` → openai; elif `OLLAMA_HOST` reachable → ollama; else `None`

JSON schema (stdlib `json`, no YAML dep):

```json
{
  "default": "anthropic",
  "providers": {
    "anthropic": {"type": "anthropic", "api_key": "${ANTHROPIC_API_KEY}", "model": "claude-haiku-4-5-20251001"},
    "local":     {"type": "openai_compat", "base_url": "http://localhost:1234/v1", "model": "llama-3.2-3b"}
  }
}
```

`${VAR}` syntax in string values is interpolated from the environment. Missing vars raise `LLMConfigError` at load time, not call time.

**Public re-exports (`llm/__init__.py`):**

```python
from .provider import LLMProviderProtocol, load_provider, LLMConfigError
from .adapters.openai import OpenAIProvider
from .adapters.anthropic import AnthropicProvider
from .adapters.openrouter import OpenRouterProvider
from .adapters.ollama import OllamaProvider
from .adapters.openai_compat import OpenAICompatProvider
```

The existing `LLMProvider` Callable alias in `classifier.py` is **not removed**. Both names coexist: `LLMProvider` (Callable, unchanged) and `LLMProviderProtocol` (the new Protocol).

### 2. `classify_smart()` (`smart.py`)

```python
def classify_smart(
    repo: str,
    *,
    llm_provider: "LLMProviderProtocol | LLMProvider | None" = None,
    confidence_threshold: float = 0.75,
    auto_load_provider: bool = True,
) -> AppDescription:
    """Rule-based first; LLM-escalated when confidence is low.

    1. Run classify() (deterministic, fast, no network).
    2. If app_category_confidence >= threshold, return the baseline.
    3. Otherwise, resolve a provider (explicit > auto-loaded > None).
    4. If no provider available, return the low-confidence baseline.
    5. Else invoke classify_agentic() and return its AppDescription.
    """

def classify_smart_async(
    repo: str,
    *,
    llm_provider: "LLMProviderProtocol | LLMProvider | None" = None,
    confidence_threshold: float = 0.75,
    auto_load_provider: bool = True,
) -> AgentClassificationResult:
    """Async variant returning the full audit trail (steps, llm_calls, iterations)."""
```

`llm_provider` accepts either the new Protocol-shaped adapters OR the existing `LLMProvider` Callable alias (since every adapter aliases `__call__ = complete`, both work).

Public re-exports added to `app_classifier/__init__.py`:
- `classify_smart`, `classify_smart_async`
- All LLM symbols from `app_classifier.llm`
- `map_code`, `CodeMap`, `FileNode`, `FunctionNode`

### 3. `map_code()` impact analysis (`app_classifier.codemap`)

```python
@dataclass
class FileNode:
    path: str                    # repo-relative
    language: str                # "python" | "js" | "ts" | "java" | "go" | "ruby" | "php"
    imports: list[str]           # repo-internal imports (resolved relative paths)
    imported_by: list[str]
    is_entry_point: bool         # True if 0 importers OR framework marker detected

@dataclass
class FunctionNode:
    file: str
    name: str
    line: int
    calls: list[str]             # "file:function" — best-effort, intra-repo only
    called_by: list[str]

@dataclass
class CodeMap:
    files: dict[str, FileNode]                 # key = path
    functions: dict[str, FunctionNode]         # key = "path:function"
    entry_points: list[str]

    def impact_of(self, target: str, *, transitive: bool = True) -> list[str]:
        """Return files (or functions) affected by changing `target`.
        `target` can be a file path or 'file:function'. Returns sorted list."""

def map_code(repo: str | Path, *, include_function_calls: bool = True) -> CodeMap: ...
```

**File-dep coverage (regex-based, intra-repo only):**

| Language | Pattern(s) |
|---|---|
| Python | `^\s*(?:from\s+(\S+)\s+import\|import\s+(\S+))` — resolve relative `.` / `..` paths |
| JS / TS | `import.*from\s+['"](\.\S+)['"]`, `require\(['"](\.\S+)['"]\)`, dynamic `import\(['"](\.\S+)['"]\)` |
| Java | `^import\s+([\w\.]+);` — match against repo package roots |
| Go | `import\s*\(\s*[\s\S]*?\)` block parser — match repo module path |
| Ruby | `require_relative\s+['"](\S+)['"]`, `require\s+['"]\./(\S+)['"]` |
| PHP | `(?:require|require_once|include|include_once)\s*\(?['"](\S+)['"]`, `use\s+([\w\\\\]+);` |

External imports (e.g., `import requests`) are noted but not added to `imports` — they're not "intra-repo."

**Function-call (Python only, this release):**

- Walk `*.py` with stdlib `ast.parse()`.
- For each `FunctionDef`/`AsyncFunctionDef`, record name + line.
- Walk `Call` nodes inside the body. Resolve to a target only when:
  - `Name` node (direct call) → match against same-file function defs
  - `Attribute` chain (`module.func()`) → match imported module + same-name function in target file
- Best-effort. Unresolved calls are dropped (no false edges).

**Entry-point detection:**

- File has `imported_by == []`, **OR**
- File contains any framework marker:
  - `if __name__ == "__main__":` (Python)
  - `@app.route`, `@router.get`, `@blueprint.route` (Python web frameworks)
  - `@RequestMapping`, `@GetMapping`, `@PostMapping` (Spring)
  - `app.get(`, `app.post(`, `router.get(`, `router.post(` (Express)
  - `func main()` (Go)
  - `class .* extends ApplicationServlet` / `extends HttpServlet` (Java servlets)

**`impact_of()` algorithm:**

- If `target` is a file path: BFS over `imported_by` (transitive=True) or one-hop (transitive=False).
- If `target` is `"file:function"`: BFS over `called_by` for function-level traversal, then fold up to the owning file. Files unrepresented in the function map (e.g., non-Python) fall back to file-level impact.
- **Cycle handling:** BFS maintains a visited-set so import cycles (common in larger codebases) don't loop. The target itself is added to visited on entry, then excluded from output.
- Output is the sorted list of impacted files (or `"file:function"` keys when traversing function-level), with the target itself excluded.

## Data flow

### `classify_smart` happy path

```
caller → classify_smart(repo)
       → classify(repo)            # deterministic baseline
       → conf >= 0.75 ? return baseline
       → load_provider()           # env / file / autodetect
       → classify_agentic(repo, llm_provider)
       → result.description
       → return to caller
```

### `map_code` flow

```
caller → map_code(repo)
       → walk repo for source files
       → file_deps.extract_imports(file) per language
       → resolve imports → build FileNode graph
       → py_calls.extract_calls(file) for *.py
       → resolve same-file + imported-name calls → FunctionNode graph
       → detect entry_points (zero in-degree + framework markers)
       → return CodeMap
```

## Error handling

| Failure mode | Behavior |
|---|---|
| No provider configured + `classify_smart` called | Returns rule-based result with low confidence. Logs a hint at `INFO` level. No exception. |
| Provider auth failure (401) | `complete()` returns `None`. `classify_agentic` records an error step and falls back to baseline. |
| Provider rate-limited (429) | One retry with 2s backoff. Then returns `None`. |
| Provider 5xx | One retry. Then returns `None`. |
| Provider returns invalid JSON for agentic schema | Existing `classify_agentic` error-handler path — recorded as error step, baseline returned. |
| `${VAR}` in config but env var missing | `LLMConfigError` raised at `load_provider()` time, not at call time. |
| `map_code` on a path that doesn't exist | `FileNotFoundError` (let stdlib raise). |
| `map_code` on a binary file | Skipped silently (caught by encoding error). |
| `ast.parse` failure on a `.py` file (syntax error) | File added to `files` but excluded from `functions`. Warning logged. |
| `impact_of("nonexistent")` | Returns `[]`. |

## Plan decomposition

The three subsystems are independent and the implementation plan SHOULD be written as three parallel-executable plan files rather than a single monolith:

- `PLAN-llm.md` — Component 1 (provider protocol + 5 adapters + config loader). Wave 1.
- `PLAN-smart.md` — Component 2 (`classify_smart` + `classify_smart_async`). Wave 2 — depends on PLAN-llm.
- `PLAN-codemap.md` — Component 3 (`map_code` + `impact_of`). Wave 1 — fully independent of LLM work.
- `PLAN-docs.md` — README + `docs/llm-providers.md` + `docs/code-mapping.md` + CHANGELOG. Wave 3 — depends on all three feature plans landing.

This lets PLAN-llm and PLAN-codemap execute concurrently. PLAN-smart waits for PLAN-llm. PLAN-docs waits for all features.

## Open assumptions (frozen unless changed)

- JSON not YAML for config
- Function-call graph Python-only this release; Java/JS/Go in v0.6.0
- `classify_smart()` returns `AppDescription`; async variant returns full `AgentClassificationResult`
- Confidence threshold default 0.75 (matches `classify_agentic`)
- All adapters use stdlib urllib — no new runtime deps. SDK adapters (optional) go in extras
- Provider `complete()` returns `str | None` (None for any failure — caller decides what to do)
- No streaming in v0.5.0 — providers always wait for the full response

## Risks

| Risk | Mitigation |
|---|---|
| Provider HTTP quirks (Anthropic vs OpenAI request shape differences) leak past the adapter boundary | Per-adapter unit tests with mocked urlopen; each adapter's request body shape pinned by snapshot test |
| Regex-based import detection produces false edges on commented-out imports | Strip line comments before regex match; accept residual false positives as low-impact (worst case: extra impacted file) |
| Python `ast.parse` on Python 3.14 source from a 3.10 venv fails on new syntax | Skip-and-log; record file in CodeMap with empty function list |
| `${VAR}` interpolation accidentally exfiltrates env vars through error messages | Never echo the substituted value in error messages — only the placeholder name |
| `map_code` on huge repos becomes a memory issue | Streaming walk (don't slurp file list); soft cap at 50k files with truncation warning |
| Doc refresh introduces drift from actual API | API reference section in README is generated from the docstrings in v0.5.1 (deferred); v0.5.0 keeps docstrings authoritative and links to them |

## Acceptance criteria

1. `pytest tests/` reports 100% pass, including ≥25 new tests covering LLM adapters, smart escalation, and code mapping.
2. `classify_smart()` against `tests/fixtures/ecommerce_django` with no provider configured returns confidence 0.95 baseline (no network).
3. `classify_smart()` against a sparse fixture (`tests/fixtures/blog_flask` with `confidence_threshold=0.99`) and a stub provider invokes the agentic path.
4. `map_code(tests/fixtures/ecommerce_django)` returns ≥1 entry point and ≥1 file-dep edge.
5. README's Quick Start three-snippet block can be copy-pasted and runs against `tests/fixtures/ecommerce_django` (or any chosen reference fixture). Each snippet must execute without error and produce the documented output.
6. `pip install app-classifier` in a fresh venv has zero non-stdlib runtime dependencies (`pip show` confirms).
7. Each of the 5 provider adapters has at least one integration-shape test (mocked HTTP).

## Out of scope / deferred to v0.6.0+

- Java / JS / Go function-call graph
- Tree-sitter-backed parsing
- Streaming completion support in provider layer
- A `compare_against_main()` helper that runs `map_code` on two commits and surfaces drift
- Provider-side caching to disk
- Cost-cap and token-cap controls per call
